Project Overview

cAIuldron is an AI-powered recipe generator that transforms ingredient photos into complete, personalized recipes using advanced machine learning. It's a production-ready full-stack system combining computer vision (CLIP + DETR), nutrition analysis (USDA FoodData Central), and large language models (fine-tuned Llama 3.2) into a seamless web interface. The system achieves 88% F1 score on ingredient detection across 525+ classes and generates high-quality recipes with 92/100 quality score while running entirely locally on consumer hardware.

System Workflow

The cAIuldron pipeline processes ingredient photos through five distinct stages, from initial upload to final recipe generation. Each stage is modular and independently testable, ensuring reliability and maintainability.

Complete pipeline workflow from photo upload to recipe generation (15-25 seconds total)

Pipeline Stages:

  • Step 1: User uploads a food/ingredient photo as input
  • Step 2: CLIP + DETR detect ingredients and output an ingredient list with bounding boxes (1-2 seconds)
  • Step 3: Map detected ingredients to USDA FoodData Central and estimate nutrition info (calories & macros) (<10ms)
  • Step 4: Generate recipes using GPT-2, Llama 3.2 1B, or Llama 3.1 8B with time field validation (1-12 seconds)
  • Step 5: Display complete recipe with ingredients, instructions, nutrition, and cooking times

Project Architecture

cAIuldron follows a modular architecture with 5 independent components that can be developed, tested, and deployed separately. The project includes comprehensive training pipelines, datasets totaling 2.23M recipes, and production-ready models optimized for consumer GPUs.

Modular project structure with 5 core components and comprehensive training pipelines

Key Components:

  • Main Application (5 modular notebooks): Model loading, ingredient detection, nutrition estimation, recipe generation, and Gradio web interface
  • Training & Development: Separate pipelines for ingredient recognition (CLIP+DETR), nutrition estimation (USDA), and recipe generation (GPT-2/Llama)
  • Data & Databases: 528 ingredients with USDA nutrition data, 525+ ingredient vocabulary, 2.23M recipe dataset from RecipeNLG
  • Trained Models: GPT-2 fine-tuned (1-2 GB), Llama 3.2 1B LoRA adapters, Llama 3.1 8B quantized GGUF (5.4 GB total)

Technical Implementation

1. Ingredient Detection Pipeline

  • Used DETR (Detection Transformer with ResNet-50 backbone) to identify regions of interest in food images
  • Implemented two-stage zero-shot classification using CLIP (ViT-Base/32) for ingredient identification across 525+ classes
  • Integrated USDA FoodData Central API for nutritional data enrichment
  • Achieved 88% F1 score and 85-95% accuracy across different ingredient categories

2. Recipe Generation with LLM Fine-tuning

  • Fine-tuned Llama 3.2 1B (1.2 billion parameters) using QLoRA (Quantized Low-Rank Adaptation)
  • Trained on consumer GPU (NVIDIA RTX 3060, 6GB VRAM) demonstrating parameter-efficient fine-tuning
  • Optimized only 2.8M parameters (0.23% of total) while keeping base model frozen
  • Curated training dataset from RecipeNLG with 7,913 high-quality recipes
  • Achieved 92/100 quality score on recipe generation evaluation metrics

3. System Architecture

  • Designed modular system with 5 independent components for scalability and maintainability
  • Implemented LLM-based fallback validation to ensure recipe completeness and correctness
  • Built Gradio web interface for seamless user interaction and real-time feedback
  • Optimized pipeline to process complete workflow in 15-25 seconds

Technical Challenges & Innovative Solutions

Building a production-ready AI recipe generator on consumer hardware presented several significant challenges. Here's how cAIuldron overcame them through innovative engineering solutions.

Challenge 1: Limited GPU Memory (6GB VRAM)

Problem: Training large language models typically requires 24GB+ VRAM, but consumer GPUs (like RTX 3060) only have 6GB, making traditional fine-tuning impossible.

Solution:

  • Implemented QLoRA (Quantized Low-Rank Adaptation) for parameter-efficient fine-tuning
  • Used 4-bit quantization to reduce model memory footprint by 75%
  • Trained only 2.8M adapter parameters (0.23%) while keeping 1.2B base model frozen
  • Achieved 92/100 quality score with <5GB VRAM usage during training

Impact: Democratized LLM fine-tuning for researchers and developers without access to expensive cloud GPUs or high-end workstations.

Challenge 2: Ingredient Diversity & Zero-Shot Detection

Problem: Food ingredients have immense variety (thousands of types), and collecting labeled training data for object detection across all categories is impractical and expensive.

Solution:

  • Leveraged CLIP's zero-shot classification capabilities (no training data needed)
  • Implemented two-stage pipeline: DETR for region detection ‚Üí CLIP for ingredient classification
  • Created structured 525+ ingredient vocabulary covering major food categories
  • Achieved 85-95% accuracy across different ingredient types without custom training

Impact: Enabled scalable ingredient detection that can easily expand to new categories by simply adding them to the vocabulary, without retraining.

Challenge 3: Recipe Quality & Time Field Accuracy

Problem: LLMs often generate incomplete recipes or hallucinate unrealistic cooking times (e.g., 5-minute roast chicken), leading to poor user experience.

Solution:

  • Curated high-quality training set: filtered 2.23M RecipeNLG recipes down to 7,913 best examples
  • Implemented regex-based time field validation for Prep Time, Cook Time, Total Time, Servings
  • Built LLM-based fallback system: if fields missing/invalid, use GPT-4 mini to estimate realistic times
  • Achieved 98%+ recipe completeness rate with accurate, validated time information

Impact: Ensured production-quality output with reliable cooking instructions, making cAIuldron suitable for real-world culinary use.

Challenge 4: System Integration & Modularity

Problem: Combining multiple AI models (CLIP, DETR, Llama) into a cohesive pipeline while maintaining code maintainability and allowing independent updates.

Solution:

  • Designed 5-module architecture: Model Loading ‚Üí Detection ‚Üí Nutrition ‚Üí Generation ‚Üí Web UI
  • Each module is independently testable and can be swapped/upgraded without affecting others
  • Created comprehensive documentation (README.md) for each pipeline component
  • Implemented Gradio interface for seamless user interaction with real-time feedback

Impact: Created a maintainable, scalable system suitable for both research experimentation and production deployment.
