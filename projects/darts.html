Project Overview

Led team in Taiwan's Ministry of Education Climate Change Innovation Competition, designing a humanity-oriented campus targeting Net Zero emissions and Vision Zero traffic fatalities. Developed the DARTS system integrating pedestrian flow and bicycle prediction models to optimize campus mobility and reduce environmental impact.

System Architecture

DARTS integrates multiple data sources and processing modules to deliver real-time campus mobility insights:

DARTS data processing pipeline and system architecture

Data Input Layer

Aggregates course selection schedules, campus bus tracking (PTX API), YouBike station status (TDX API), and parking lot occupancy data through unified interface.

Campus Numeric Route Model

Digitizes campus road network with nodes (buildings) and edges (paths), incorporating slope gradients, distance, and surface quality as base parameters for route optimization.

Equivalent Route Model

Transforms numeric model using equivalent route formula, converting physical distances into perceived effort by factoring in elevation changes, weather conditions, and user preferences.

DARTS Kernel Strategy Module

Core prediction engine using ARIMA (1,1,0) time series models to forecast bike availability, parking occupancy, and pedestrian flow 10 minutes ahead with 86.1% accuracy.

Technical Implementation

  • ARIMA Time Series Prediction: Implemented ARIMA (1,1,0) models for 12 YouBike stations within 1km radius, predicting availability 10 minutes ahead with 86.1% accuracy
  • TDX API Integration: OAuth 2.0 authentication with Taiwan Transport Data Exchange (TDX) for real-time YouBike 2.0 station data, supporting 50 calls/second with unlimited daily requests
  • Campus Numeric Route Model: Digitized NSYSU road network with nodes (buildings) and edges (paths), incorporating slope gradients, distances, and surface quality for route optimization
  • Equivalent Route Formula: Transformed physical distances into perceived effort by factoring elevation changes, weather, and user preferences for human-centric route recommendations
  • Automated Data Collection: Python-based system collecting data every 10 minutes (600 seconds), appending to cumulative CSV storage, requiring minimum 50 records before predictions begin
  • Multi-Source Data Fusion: Integrated course schedules, PTX public transit API, YouBike TDX API, and parking sensors into unified DARTS Kernel Strategy module for holistic mobility insights

Technical Challenges & Solutions

‚ö° Real-Time Data Accuracy

API latency and data inconsistencies across multiple sources (TDX, PTX, parking sensors) made it difficult to provide reliable real-time predictions.

Solution:

Implemented cumulative data storage with 10-minute intervals and ARIMA warm-up period (50 records) to ensure prediction stability and accuracy before going live.

‚ö° Route Optimization Complexity

Traditional GIS shortest-path algorithms don't reflect real user experience - a steep uphill path may be shorter but feels much longer to pedestrians.

Solution:

Developed Equivalent Route Formula converting physical distances into perceived effort by incorporating slope gradients, weather conditions, and user preferences for human-centric routing.

‚ö° Stakeholder Coordination

Aligning interests of university administration (policy goals), students (convenience), and technical team (feasibility) while maintaining data privacy compliance.

Solution:

Established collaborative framework with Student Association as liaison, using de-identified data, and phased rollout approach to build trust and gather feedback incrementally.

‚ö° Low-Cost Hardware Implementation

Campus-wide sensor deployment (VMS boards, parking detection, traffic monitoring) would require significant infrastructure investment beyond project budget.

Solution:

Prioritized software-based solutions leveraging existing public APIs and campus data sources, proposing hardware additions as future enhancements only where critical for accuracy.
